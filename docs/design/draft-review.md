# Draft Review Workflow Design

## Overview

Drafts generated by the AI Extractor are stored in a `pending` state. Users review them through a REST API, choosing to confirm (convert to a real entity), edit (modify fields before confirming), or discard.

## Draft Status Machine

```
pending â†’ confirmed
pending â†’ discarded
```

MVP keeps this simple: no `edited` state. Edit + confirm is a single action.

## REST API

### List pending drafts

```
GET /api/drafts?status=pending
```

Returns an array of draft objects. Default status filter is `pending`.

### Get single draft

```
GET /api/drafts/{id}
```

Returns the draft with all fields.

### Review a draft

```
POST /api/drafts/{id}/review
```

Request body:

```json
{
  "action": "confirm",
  "updates": {
    "title": "modified title",
    "description": "modified description"
  }
}
```

- `action`: `confirm` or `discard` (required).
- `updates`: optional field overrides applied before confirming.

### Responses

| Action | Status | Body |
|--------|--------|------|
| confirm | 200 | The created entity object |
| discard | 200 | The updated draft with status `discarded` |
| invalid action | 400 | Error JSON |
| draft not found | 404 | Error JSON |
| draft not pending | 409 | Error JSON |

## Entity Generation Rules

When a draft is confirmed, a new entity is created in the corresponding table based on `draft_type`:

| draft_type | Target table | Field mapping |
|------------|-------------|---------------|
| `task` | `family_tasks` | titleâ†’title, descriptionâ†’description, due_atâ†’due_at |

| `note` | `family_notes` | titleâ†’title, descriptionâ†’content, topicâ†’topic |

The draft record is updated: `status` â†’ `confirmed`, `entity_id` is set to the new entity's ID.

This operation must be atomic (within a single SQLite transaction).

## Edge Cases

- Reviewing a non-pending draft returns 409.
- Confirming with empty updates uses the original draft fields.
- Discarding does not create any entity.
- Multiple confirm calls on the same draft: second call returns 409.
