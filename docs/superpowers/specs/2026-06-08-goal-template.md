## 阶段一：后端开发

在 `apps/api/` 目录下，基于现有 Go 代码扩展。

### Step 1: 数据库迁移

在 `store.go` 的 `migrate()` 中新增以下表（不影响现有表）：
- `family_members`
- `finance_accounts`
- `finance_records`
- `health_profiles`
- `health_records`
- `work_status`
- `work_records`
- `family_status`
- `family_records`
- `knowledge_notes`
- `conversations`
- `messages`

字段定义严格按照设计文档。所有表都要写好。

### Step 2: Store 层 CRUD

为每张新表实现完整的 CRUD 操作。遵循现有代码风格：
- 每个操作用 `s.mu.Lock()` 保护写操作
- 使用参数化 SQL（`?` 占位符）
- 查询返回 slice 或指针+error

### Step 3: API Handler

在 `server.go` 中注册新路由，新建 handler 文件：
- `member_handlers.go` — 家庭成员 CRUD
- `domain_handlers.go` — 四领域（财务/健康/工作/家庭）的状态+流水 CRUD
- `note_handlers.go` — 知识笔记 CRUD
- `chat_handlers.go` — AI 对话（创建对话、发消息、获取历史）
- `config_handlers.go` — AI 配置读写
- `quick_entry_handlers.go` — 自然语言快速录入

所有 `/api/` 路由需要 Bearer token 认证（复用现有逻辑）。

### Step 4: Agent 系统

新建 `agent.go`，实现：
- `Agent` 接口（Name, Domain, SystemPrompt, RetrieveContext）
- `ButlerAgent` — 路由 + 综合
- `FinanceAgent` — 财务领域
- `HealthAgent` — 健康领域
- `WorkAgent` — 工作领域
- `FamilyAgent` — 家庭事务领域

每个 Agent 的 RetrieveContext 查三层：状态表 → 流水表（近10条）→ 知识笔记表。

管家 Agent 的工作流：
1. 调 LLM 分析用户问题，判断涉及哪些领域
2. 并发调对应专业 Agent（goroutine + errgroup）
3. 综合（多领域时再调一次 LLM 汇总）
4. 返回结果

### Step 5: LLM 调用封装

扩展 `ai_extractor.go` 中已有的 OpenAI 兼容调用模式：
- 支持自定义 system prompt
- 支持上下文注入
- 支持对话历史
- 支持流式响应（SSE）

### Step 6: 自然语言快速录入

实现 `POST /api/quick-entry`：
1. 接收自然语言输入
2. 调 LLM 拆分为多条领域记录草稿
3. 返回草稿给前端确认
4. 确认后写入对应流水表 + 更新状态表

### Step 7: 为每个新功能写测试

为以下内容编写 `*_test.go`：
- Store 层 CRUD（每张新表）
- Agent 路由逻辑
- LLM 调用（mock）
- API handler（HTTP 测试）

运行 `cd apps/api && go test ./...` 确保全部通过。

## 阶段二：前端开发

在 `apps/web/` 目录下，使用 Vue 3 + Vite。

### Step 8: 初始化项目

```bash
cd apps
npm create vite@latest web -- --template vue
cd web
npm install vue-router@4 axios
```

### Step 9: 页面实现

按优先级实现：

1. **设置页** (`/settings`) — AI 配置（endpoint、model、API key）
2. **成员页** (`/members`) — 家庭成员 CRUD
3. **仪表盘** (`/`) — 状态卡片 + 快速录入 + 最近记录
4. **对话页** (`/chat`) — 与管家 AI 对话（支持流式）
5. **财务页** (`/finance`) — 账户余额 + 收支流水
6. **健康页** (`/health`) — 成员健康状态 + 指标记录
7. **工作页** (`/work`) — 工作状态 + 项目/deadline
8. **家庭页** (`/family`) — 家庭状态 + 日程/家务
9. **笔记页** (`/notes`) — 知识笔记浏览/搜索

路由、导航栏、响应式布局。中文界面。

### Step 10: API 对接

封装 axios 实例，所有请求带 Bearer token。对接后端全部接口。

## 阶段四：验收测试

### Step 12: 启动服务


确认服务在配置的端口启动成功。

### Step 13: 领域测试 — 财务 (10题)

依次通过对话 API 发送以下问题，验证回答准确性：

1. "家里现在一共有多少钱？" → 应回答总资产约 -967300（含负债）或正资产约 710700
2. "这个月花了多少？跟上个月比呢？" → 应列出6月支出 vs 5月支出
3. "房贷还剩多少？每个月还多少？" → 168万，月供6500
4. "家庭净资产大概是多少？" → 正资产 - 负债
5. "这个月最大的开支是什么？" → 应该是房贷或教育
6. "爷爷最近有什么医疗支出吗？" → 320药费
7. "我们每月固定支出大概是多少？" → 房贷6500+车贷2800+定投8000
8. "基金投了多少钱？" → 稳健35万+指数18万=53万
9. "今年上半年收入大概多少？" → 应汇总4-6月工资+奖金
10. "有什么可以省钱的地方吗？" → AI应给出合理建议

每个问题检查：
- [ ] 返回了回答（非空）
- [ ] 数字基本正确（允许四舍五入差异）
- [ ] 没有幻觉（编造不存在的数据）
- [ ] 中文自然流畅

3. "志豪最近忙不忙？工作和健康怎么样？" → 工作忙+血压略高
4. "一鸣最近各方面情况怎么样？" → 健康(过敏+视力)+学习(兴趣班)+日程(家长会)
5. "帮我看一下下个月需要提前准备什么" → 迁移评审、暑假看护、外婆生日、出游
